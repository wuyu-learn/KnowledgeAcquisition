# Changelog

## 项目概况

`信披-定向知识获取` 是一个面向公募基金定期报告（PDF）的定向信息提取工具前端 demo。

目标是通过 AI 自动从 PDF 定期报告中抽取指定章节内容，降低人工检索与汇总成本。当前阶段为纯静态前端原型，所有数据均为 mock 数据，用于演示页面结构与交互流程。

---

## 当前已实现功能

### 1. 页面整体结构

- 顶部导航栏（深蓝顶栏 + 系统标签栏）
- 筛选区
- 结果列表区
- 分页组件
- 操作日志弹窗
- 批量查询日志弹窗
- 定时任务配置弹窗
- 正在执行任务面板（Task Panel）

### 2. 筛选区

筛选项：

- 报告类型（季度报告 / 中期报告 / 年度报告）
- 知识获取类型（10 种可选类型）
- 报告期（近 5 年，按时间倒序平铺）

操作按钮：

- 定时任务配置
- 查询
- 重置
- 导出日志

### 3. 结果列表区

表格字段：

- 勾选框
- 序号
- 报告类型
- 知识获取类型
- 报告期
- 任务状态（提取成功 / 提取失败）
- 任务创建时间
- 最后修改时间
- 操作人
- 操作（下载 / 删除 / 日志 / 重试）

工具栏：

- 已选择 x 条
- 批量导出（勾选 ≥2 条时可用）
- 批量查询日志（勾选 ≥2 条时可用）
- 刷新

### 4. 操作日志弹窗

- 单行日志弹窗：操作时间、操作人、操作类型、任务状态
- 批量查询日志弹窗：序号、报告类型、知识获取类型、报告期、操作人、操作时间、操作类型

日志事件模型：

```js
{
  time: string,      // 日志时间
  operator: string,  // 操作人（系统 / 用户）
  event: string,     // task_created | ai_extract_completed
  status: string     // success | failed | running
}
```

### 5. 定时任务配置

- 弹窗展示未来 3 个待执行定时任务
- 支持启用 / 停用开关
- 支持修改执行时间（datetime-local）
- 保存 / 取消操作

### 6. 正在执行任务面板（Task Panel）

触发条件：

- 点击“重试”按钮后

展示内容：

- 报告类型、知识获取类型、报告期
- 公告处理进度（processed / total / 百分比 / 进度条）
- 已用时间、预计剩余时间
- 当前排队任务数量
- 查看详情、取消任务按钮

任务模拟规则：

- 进度从 0% 自动增长到 100%
- 完成后任务状态变为 success
- 自动隐藏 Task Panel
- 结果列表中对应失败任务会更新为“提取成功”
