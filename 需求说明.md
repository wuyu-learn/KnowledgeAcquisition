## 一、需求背景

当前业务人员需人工从公募基金定期报告PDF中查找指定章节，并复制粘贴相关内容进行汇总分析，工作量大且效率较低。本功能通过AI自动提取定期报告中的目标信息，并提供后台页面对取数任务及提取结果进行统一管理，降低人工操作成本，提高信息获取效率。

## 二、页面功能
页面主要由任务管理和结果筛选及展示两部分组成。

**任务管理**：提供信息提取任务的统一管理，支持查看任务执行状态、执行进度及执行记录，并支持手动发起任务、配置定时任务执行时间等操作。
**结果筛选及展示**：支持按多种条件筛选信息提取结果，并以表格形式展示，支持导出、删除等操作，便于业务人员进行信息复核。

## 三、项目目录说明

```text
demo/
│
├── index.html          // 页面主文件，负责整体页面结构及内容布局
│
├── css/
│   ├── style.css       // 页面样式，包含页面布局及业务样式
│   └── component.css   // 公共组件样式，如按钮、表格、弹窗、Tag、表单等
│
├── js/
│   ├── data.js         // Mock数据，统一维护页面演示所需的模拟数据
│   └── app.js          // 页面交互逻辑，如Tab切换、弹窗、筛选、按钮事件等
│
└── assets/（按需创建）
    ├── images/         // 图片资源
    ├── icons/          // 图标资源
    └── fonts/          // 字体资源
```

**说明：** `assets` 目录用于存放静态资源，当页面涉及图片、图标或自定义字体时按需创建；若无相关资源，可不创建该目录。
